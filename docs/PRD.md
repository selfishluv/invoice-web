# 아이타임 (AiTime) MVP PRD

## 핵심 정보

**목적**: Notion DB를 CMS로 활용하여 비개발자(부모)도 콘텐츠를 쉽게 관리하고, 웹에서 아이의 성장 기록을 타임라인 형태로 볼 수 있게 한다
**사용자**: 아이의 성장 기록을 Notion으로 관리하고 싶은 부모 (비개발자 포함)

---

## 사용자 여정

```
1. 타임라인 페이지 (메인)
   - 사이트 접속 시 자동으로 타임라인 페이지 표시
   - 전체 성장 기록이 날짜 역순으로 카드 형태로 나열됨
   ↓ 태그 칩 클릭

2. 태그 필터 적용 상태
   - 선택한 태그에 해당하는 기록만 표시
   - 여러 태그 동시 선택 가능
   - 태그 재클릭 시 필터 해제
   ↓ 기록 카드 클릭

3. 상세 보기 모달
   - 사진 확대 + 전체 메모 표시
   - 모달 닫기 → 타임라인으로 복귀
   ↓ 모달 닫기

4. 타임라인 페이지 복귀
   - 기존 필터 상태 유지
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명             | 설명                                                    | MVP 필수 이유                | 관련 페이지                |
| -------- | ------------------ | ------------------------------------------------------- | ---------------------------- | -------------------------- |
| **F001** | Notion 데이터 연동 | Notion API를 통해 DB의 성장 기록을 fetch하여 렌더링     | 서비스의 데이터 소스 핵심    | 타임라인 페이지            |
| **F002** | 타임라인 렌더링    | 날짜순 정렬된 성장 기록 카드를 세로 타임라인으로 표시   | 서비스 핵심 UI               | 타임라인 페이지            |
| **F003** | 태그 필터링        | 상단 태그 칩(chip) UI로 원하는 태그의 기록만 필터링     | 기록 탐색 핵심 UX            | 타임라인 페이지            |
| **F004** | 사진 모달 뷰어     | 카드 클릭 시 사진 확대 및 전체 메모 표시 모달           | 기록 상세 확인 핵심 기능     | 타임라인 페이지, 상세 모달 |
| **F005** | 요약 통계 표시     | 총 기록 수, 첫 이벤트 날짜, 최근 기록 등 상단 요약 정보 | 성장 기록 전체 맥락 제공     | 타임라인 페이지            |
| **F006** | ISR 성능 최적화    | Incremental Static Regeneration으로 Notion 데이터 캐싱  | API 호출 최소화 및 빠른 로딩 | 타임라인 페이지            |

## 메뉴 구조

```
아이타임 (AiTime) 내비게이션
└── 헤더
    ├── 로고/사이트명 (타임라인 페이지로 이동)
    └── (추가 메뉴 없음 - 단일 페이지 서비스)

타임라인 페이지 내 인터랙션
├── 태그 필터 칩 영역 - F003 (태그 필터링)
│   ├── 전체 보기 칩
│   ├── 첫걸음 칩
│   ├── 첫말 칩
│   ├── 첫니 칩
│   ├── 생일 칩
│   ├── 병원 칩
│   └── 여행 칩 (Notion DB에 등록된 태그 동적 렌더링)
├── 요약 통계 섹션 - F005 (요약 통계 표시)
└── 타임라인 카드 목록 - F001, F002 (Notion 연동 + 타임라인 렌더링)
    └── 카드 클릭 → 상세 모달 열기 - F004 (사진 모달 뷰어)
```

## 페이지별 상세 기능

| **역할**        | 서비스의 메인이자 유일한 페이지로, 아이의 성장 기록 전체를 타임라인 형태로 표시                                                                                                                                                                                                                                                                                                                                                       |

## 데이터 모델

### GrowthRecord (Notion DB 매핑)

| 필드      | 설명             | 타입/관계         |
| --------- | ---------------- | ----------------- |
| id        | Notion 페이지 ID | string (UUID)     |
| title     | 이벤트 제목      | string            |
| date      | 이벤트 발생 날짜 | string (ISO 8601) |
| photoUrl  | 외부 이미지 URL  | string (nullable) |
| tags      | 이정표 태그 목록 | string[]          |
| memo      | 상세 메모/기록   | string            |
| ageMonths | 개월 수          | number (nullable) |

### Notion DB 스키마 (실제 설정값)

| Notion 속성명 | Notion 타입         | 설명                                    |
| ------------- | ------------------- | --------------------------------------- |
| Title         | title               | 이벤트 제목 (필수)                      |
| Date          | date                | 이벤트 발생 날짜 (필수)                 |
| PhotoURL      | url                 | 외부 이미지 URL                         |
| Tags          | multi_select        | 첫걸음, 첫말, 첫니, 생일, 병원, 여행 등 |
| Memo          | rich_text           | 상세 메모/기록                          |
| Age           | formula 또는 number | 개월 수                                 |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 15.5.3** (App Router + Turbopack) - React 풀스택 프레임워크
- **TypeScript 5** - 타입 안전성 보장
- **React 19.1.0** - UI 라이브러리 (최신 동시성 기능)

### 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진) - 유틸리티 CSS 프레임워크
- **shadcn/ui** (new-york style) - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### CMS & 데이터

- **Notion API** (`@notionhq/client`) - 콘텐츠 관리 시스템 (CMS)
- **ISR (Incremental Static Regeneration)** - Notion 데이터 캐싱 및 성능 최적화

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼

### 패키지 관리

- **npm** - 의존성 관리

## 구현 순서

1. **Notion API 연동**: `@notionhq/client` 설치, 환경 변수 설정, DB fetch 유틸 함수 작성
2. **타임라인 페이지 UI**: ISR 적용 서버 컴포넌트로 기록 카드 목록 렌더링
3. **태그 필터 + 사진 모달**: 클라이언트 컴포넌트로 필터링 상태 관리 및 모달 구현
